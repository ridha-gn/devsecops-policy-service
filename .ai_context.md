# DevSecOps Policy-as-Code Service

## Project Overview
This project is a DevSecOps Policy-as-Code service that analyzes infrastructure code (Terraform, Dockerfile, Kubernetes YAML) and enforces security policies. It consists of a FastAPI backend, an enforcement engine with 103 security policies, a VS Code extension for developer feedback, and Grafana dashboards for monitoring and visualization.

## Key Components

### 1. Backend API (FastAPI)
- **Path**: `api/` and `main.py`
- **Purpose**: Provides endpoints for scanning files, fetching scan history, retrieving compliance metrics, generating reports, and exporting Prometheus metrics.
- **Key Files**: 
  - `main.py`: Application entry point.
  - `api/routes.py`: Contains API endpoints (`/scan`, `/fix`, `/metrics`, `/history`, etc.).
  - `api/prometheus.py`: Prometheus metrics configuration.

### 2. Policy Engine
- **Path**: `engine/`
- **Purpose**: Parses infrastructure files, normalizes them, and evaluates them against 103 built-in security policies. It also includes an auto-fix mechanism.
- **Key Files**:
  - `engine/parser.py`: Parses Terraform, YAML, and Dockerfiles.
  - `engine/normalizer.py`: Normalizes parsed configurations for policy evaluation.
  - `engine/policy_engine.py`: Runs the normalized code against security policies.
  - `engine/auto_fix.py`: Automatically remediates policy violations.
  - `engine/policies/security_policies.py`: Contains 103 built-in security policies.

### 3. VS Code Extension
- **Path**: `vscode-extension/`
- **Purpose**: Integrates the DevSecOps Policy Service directly into the IDE.
- **Features**: 
  - Scans Terraform, Dockerfiles, and YAML files on save or via command.
  - Offers auto-fix functionality directly in the editor.
  - Connects to the local FastAPI service (default: `http://localhost:8000`).

### 4. Dashboard & Monitoring
- **Path**: `dashboard.html`, `config/grafana_dashboard.json`, `config/prometheus.yml`
- **Purpose**: Visualizes security scanning results, violations, and system metrics.
- **Components**:
  - HTML/JS Dashboard: `dashboard.html` visualizes scan history and metrics.
  - Grafana & Prometheus: Used for detailed operational and security metrics.

### 5. LaTeX Report
- **Path**: `rapport/`
- **Purpose**: The user's End of Studies (PFE) technical report documenting the project theory, architecture, policies, and implementation sprints.

## Tech Stack
- **Python**: FastAPI, Pydantic, Uvicorn, pytest, python-hcl2, pyyaml.
- **Node.js**: VS Code Extension API.
- **Monitoring**: Prometheus, Grafana.
- **Documentation**: LaTeX.

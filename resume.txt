WHAT THE PROJECT DOES
A microservice that automatically scans infrastructure code (Terraform, Dockerfile, Kubernetes YAML) for security violations BEFORE deployment. It enforces 100 security policies, responds in ~46ms average, and blocks insecure code in Jenkins CI/CD pipelines. It provides detailed explanations of WHY code is insecure and HOW to fix it.
Key concept: "Shift-left security" — moving security checks from post-deployment to pre-deployment.

THE PROBLEM IT SOLVES
Organizations deploying cloud infrastructure face critical security risks. Developers introduce vulnerabilities that reach production causing data breaches. Manual security reviews take 2-3 days and don't scale. Real incidents:

Capital One (2019): Public S3 bucket → 100M+ records leaked → $80M fine
Uber (2016): Exposed database → 57M records stolen → $148M settlement
Toyota (2023): Leaked credentials → 10 years of customer data exposed

4-LAYER ARCHITECTURE
Layer 1 — API Layer (FastAPI):

POST /analyze — scan infrastructure code
GET /metrics — real-time statistics
GET /health — service health check
GET /history — scan history from SQLite
GET /report/{filename} — HTML security report

Layer 2 — Parser Layer:

TerraformParser: uses python-hcl2
DockerfileParser: custom regex-based
KubernetesParser: uses PyYAML

Layer 3 — Normalizer Layer:

Converts all 3 formats to a common Resource model
Without normalizer: 100 policies × 3 formats = 300 implementations
With normalizer: 100 policies × 1 format = 100 implementations
Result: 67% reduction in code complexity

Layer 4 — Policy Engine:

Runs all 100 policies
Applies severity threshold filter
Makes ALLOW/BLOCK decision
Returns violations with recommendations
